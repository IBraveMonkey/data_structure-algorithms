## 2. ğŸš‘ Fixing Mistakes (Undoing Actions)

### 2.1. Amend the Last Commit
`git commit --amend`
Allows you to add a forgotten file or fix a typo in the most recent commit message. Note that this changes the commit's hash!

### 2.2. Discard Changes in a File
`git restore <file>` (modern) or `git checkout -- <file>` (legacy).
Reverts a file back to its state in the last commit. All unsaved local changes will be permanently lost.

### 2.3. Reverting with Reset
- `--soft`: Removes the commit but **keeps your changes in the Staging Area**. Useful for "re-doing" a commit.
- `--mixed` (default): Removes the commit and the staging area entries, but keeps the files on your disk.
- `--hard`: **Deletes everything**. It removes the commit, clears the staging area, and deletes the actual file changes.

> [!CAUTION]
> `git reset --hard` is a destructive operation. Use it only if you are 101% certain you want to lose your work.

## 3. ğŸ§­ Advanced Navigation

### 3.1. Reflog â€” The Time Machine
`git reflog`
Git records every change to the `HEAD` pointer. If you accidentally delete a branch or perform a `reset --hard` to the wrong place, `reflog` will show you the commit hash from before the error occurred. You can then use `git checkout <hash>` to recover your code.

### 3.2. Cherry-pick â€” Selective Copying
`git cherry-pick <hash>`
Allows you to copy one specific commit from any branch into your current active branch.

### 3.3. Stash â€” Temporary Storage
- `git stash`: "Pockets" your current uncommitted changes to give you a clean working directory.
- `git stash pop`: Brings those changes back.
Useful for when you need to switch branches urgently but aren't ready to commit your current work.

### 3.4. Bisect â€” Finding the Culprit
`git bisect`
A binary search tool used to find the specific commit that introduced a bug. You mark a "good" (working) commit and a "bad" (broken) commit, and Git guides you through the history until the source of the crash is identified.
